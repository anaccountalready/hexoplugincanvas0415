# Hexo Canvas Doodle 新增功能说明文档

## 文档版本
- 版本：v2.0.0
- 日期：2026-04-18
- 新增功能：无限画布、迷你地图导航

---

## 一、功能概述

本次更新为 Hexo Canvas Doodle 插件新增了两大核心功能：

1. **♾️ 无限画布（Infinite Canvas）**：打破固定尺寸限制，画布可以根据绘制内容自动扩展，支持任意方向的无限绘制。

2. **🗺️ 迷你地图导航（Minimap Navigation）**：在画布右下角显示实时缩略图，支持点击定位和拖拽导航，快速浏览整个画布内容。

---

## 二、无限画布功能详解

### 2.1 功能描述

传统的 Canvas 元素有固定的尺寸限制，当绘制内容超出画布边界时会被裁剪。无限画布功能解决了这一问题，使用户可以在任意方向自由绘制，无需担心边界限制。

### 2.2 技术实现

#### 2.2.1 离屏 Canvas 架构

```
┌─────────────────────────────────────────────────────────────┐
│                     显示 Canvas (Display Canvas)              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                    视口区域 (Viewport)               │    │
│  │                                                     │    │
│  │   只渲染当前可见区域，通过 CSS transform 实现平移缩放   │    │
│  │                                                     │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ 同步
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    离屏 Canvas (Offscreen Canvas)            │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              存储所有绘制内容                         │    │
│  │  - 初始大小：2000 x 2000 像素                       │    │
│  │  - 动态扩展：需要时自动翻倍扩展                       │    │
│  │  - 支持负坐标区域                                    │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

#### 2.2.2 核心函数说明

| 函数名 | 功能描述 | 文件位置 |
|--------|----------|----------|
| `ensureCanvasSize(x, y)` | 检查坐标是否超出当前离屏canvas边界，需要时自动扩展 | `canvas-doodle.js:64-105` |
| `updateContentBounds(x, y)` | 更新内容边界记录，用于迷你地图和保存时的裁剪 | `canvas-doodle.js:107-123` |
| `syncToDisplayCanvas()` | 将离屏canvas的可见区域同步到显示canvas | `canvas-doodle.js:125-154` |

#### 2.2.3 扩展策略

- **边界阈值**：当绘制位置距离当前边界小于 500 像素时触发扩展检查
- **扩展方式**：每次将画布尺寸翻倍（宽度和高度各乘以 2）
- **负坐标支持**：当绘制到 x < 0 或 y < 0 区域时，自动创建新空间并偏移原有内容
- **偏移处理**：扩展时自动调整内容边界记录，保持坐标一致性

### 2.3 使用示例

```javascript
// 用户无需额外操作，无限画布功能自动生效
// 只需正常绘制，系统会自动处理边界扩展

// 场景1：向右绘制超出初始画布
// - 初始画布：2000px 宽度
// - 绘制到 x > 1500px 时触发检查
// - 扩展为 4000px 宽度

// 场景2：向左绘制到负坐标
// - 平移画布后绘制到 x < 0 区域
// - 自动扩展并偏移原有内容
// - 所有坐标自动保持一致
```

### 2.4 保存与加载优化

- **保存时**：只裁剪有内容的区域（带 50px 内边距），减少存储占用
- **加载时**：将图片居中绘制到离屏canvas，并自动更新内容边界

## 三、迷你地图导航功能详解

### 3.1 功能描述

迷你地图是一个位于画布右下角的实时缩略图，提供以下功能：

1. **实时预览**：显示整个画布的内容缩略图
2. **视口指示**：用红色矩形标记当前可见区域在整个画布中的位置
3. **点击定位**：点击迷你地图任意位置，画布视图自动跳转到对应区域
4. **拖拽导航**：在迷你地图上拖拽可以快速浏览整个画布

### 3.2 技术实现

#### 3.2.1 架构图

```
┌──────────────────────────────────────────────────────────┐
│                    主画布视口 (800x400)                   │
│  ┌────────────────────────────────────────────────────┐  │
│  │                                                    │  │
│  │              当前可见区域                           │  │
│  │                                                    │  │
│  │                                                    │  │
│  │                                                    │  │
│  │                                                    │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│                              ┌─────────────────────────┐ │
│                              │   🗺️ 迷你地图 (150x150) │ │
│                              │  ┌───────────────────┐  │ │
│                              │  │                   │  │ │
│                              │  │  整个画布缩略图    │  │ │
│                              │  │                   │  │ │
│                              │  │  ┌─────┐          │  │ │
│                              │  │  │红色 │ ← 视口    │  │ │
│                              │  │  │矩形 │   指示器  │  │ │
│                              │  │  └─────┘          │  │ │
│                              │  └───────────────────┘  │ │
│                              └─────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```

#### 3.2.2 核心函数说明

| 函数名 | 功能描述 | 文件位置 |
|--------|----------|----------|
| `initMinimap()` | 初始化迷你地图canvas，绑定事件监听器 | `canvas-doodle.js:156-186` |
| `getContentBounds()` | 获取内容边界（用于迷你地图的智能裁剪） | `canvas-doodle.js:188-205` |
| `updateMinimap()` | 更新迷你地图显示，绘制内容缩略图和视口指示器 | `canvas-doodle.js:207-271` |
| `navigateMinimap(e)` | 根据迷你地图上的点击/拖拽位置导航画布视图 | `canvas-doodle.js:291-320` |

#### 3.2.3 迷你地图参数配置

```javascript
// 在 canvas-doodle.js 顶部定义的常量
const MINIMAP_SIZE = 150;      // 迷你地图尺寸（正方形）
const MINIMAP_PADDING = 10;    // 迷你地图与视口边缘的间距
```

### 3.3 智能边界裁剪

迷你地图采用智能边界裁剪算法，只显示有实际内容的区域：

1. **无内容时**：显示整个离屏canvas区域
2. **有内容时**：
   - 计算所有非透明像素的边界（minX, minY, maxX, maxY）
   - 添加 100px 内边距
   - 保持在离屏canvas范围内
   - 按比例缩放到迷你地图尺寸

### 3.4 交互说明

| 操作 | 效果 |
|------|------|
| 鼠标悬停 | 迷你地图边框变为蓝色，不透明度提升 |
| 左键点击 | 画布视图跳转到点击位置（以点击位置为中心） |
| 左键拖拽 | 画布视图跟随拖拽位置实时更新 |
| 画布平移/缩放 | 迷你地图中的视口指示器实时更新位置和大小 |

迷你地图的样式定义在 `canvas-doodle.css` 中：

## 六、使用指南

### 6.1 基础操作

#### 6.1.1 绘制操作

```
操作方式：鼠标左键拖拽
效果：在画布上绘制线条
注意：绘制到边界附近时画布会自动扩展，无需手动操作
```

#### 6.1.2 平移画布

```
方式一：按住鼠标中键拖拽
方式二：按住空格键 + 左键拖拽
方式三：点击控制栏的方向按钮（↑↓←→）
```

#### 6.1.3 缩放画布

```
方式一：鼠标滚轮（以鼠标位置为中心缩放）
方式二：点击控制栏的 +/- 按钮
方式三：点击"重置"按钮恢复 100% 缩放
```

### 6.2 迷你地图操作

#### 6.2.1 查看缩略图

迷你地图默认显示在画布右下角，实时展示所有绘制内容。

#### 6.2.2 快速定位

1. 观察迷你地图中的红色矩形（表示当前视口）
2. 点击迷你地图上的目标位置
3. 画布视图自动跳转到对应位置

#### 6.2.3 拖拽导航

1. 在迷你地图上按住左键
2. 拖拽移动（红色矩形跟随移动）
3. 松开左键完成导航

### 6.3 无限画布特性

#### 6.3.1 向右/向下扩展

- 正常绘制即可，系统自动检测边界
- 每次扩展将画布尺寸翻倍（2000 → 4000 → 8000...）

#### 6.3.2 向左/向上扩展（负坐标）

1. 使用平移功能将画布向右/向下移动
2. 在左侧/上方的"空白"区域绘制
3. 系统自动创建负坐标区域

#### 6.3.3 保存与加载

- **保存**：只保存有内容的区域，自动裁剪空白
- **加载**：图片自动居中，内容边界自动更新

## 七、性能优化建议

### 7.1 内存管理

- 离屏canvas初始大小：2000 x 2000 像素（约 16MB 内存）
- 每次扩展翻倍，建议控制在合理范围内
- 定期清除不需要的内容

### 7.2 渲染优化

- `syncToDisplayCanvas()` 只渲染可见区域
- 迷你地图更新频率与画布操作同步
- 使用 CSS transform 实现高效的平移缩放

### 7.3 保存优化

- 保存时只裁剪有内容的区域
- 使用 PNG 格式，支持透明度
- 状态信息单独存储在 localStorage

## 十、已知限制与未来规划

### 10.1 当前限制

1. **离屏canvas最大尺寸**：受浏览器限制，通常为 16384 x 16384 像素
2. **内存占用**：大尺寸画布会占用较多内存
