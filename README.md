# hexo-canvas-doodle

一个用于Hexo博客的画布涂鸦插件，允许在文章中嵌入可交互的画布，支持实时涂鸦和本地保存功能。

## 功能特性

- 🎨 **简单易用**：只需在文章中添加一个标签即可嵌入画布
- ✏️ **黑色画笔**：提供简洁的黑色画笔涂鸦功能
- 💾 **本地保存**：涂鸦内容自动保存到浏览器本地存储
- 📱 **响应式**：支持鼠标和触摸设备操作
- 🔄 **实时加载**：随时可以加载之前保存的涂鸦内容

## 安装

### 方法一：npm安装（推荐）

```bash
npm install hexo-canvas-doodle --save
```

### 方法二：本地安装

1. 将插件文件复制到Hexo博客的 `node_modules/hexo-canvas-doodle` 目录下
2. 在博客根目录的 `_config.yml` 中添加：

```yaml
plugins:
  - hexo-canvas-doodle
```

## 使用方法

在文章中使用 `{% canvas %}` 标签即可嵌入画布：

```markdown
{% canvas my-canvas 800 400 %}
```

### 参数说明

- **第一个参数**：画布ID（可选，默认自动生成）
- **第二个参数**：画布宽度（可选，默认800px）
- **第三个参数**：画布高度（可选，默认400px）

### 示例

```markdown
# 我的涂鸦文章

这是一个简单的画布示例：

{% canvas demo-canvas 600 300 %}

你可以在上面随意涂鸦，点击保存按钮将内容保存到本地。
```

## 控制按钮

每个画布下方都有三个控制按钮：

- **清除**：清空当前画布内容
- **保存**：将当前画布内容保存到浏览器本地存储
- **加载**：从本地存储加载之前保存的画布内容

## 技术实现

### 项目结构

```
hexo-canvas-doodle/
├── index.js              # 插件主入口文件
├── package.json          # 项目配置文件
├── assets/
│   ├── canvas-doodle.css  # 画布样式文件
│   └── canvas-doodle.js   # 画布交互逻辑
├── README.md             # 项目说明文档
└── USER_GUIDE.md         # 用户手册
```

### 核心文件说明

#### 1. index.js（插件主入口）

- **标签注册**：使用 `hexo.extend.tag.register` 注册 `canvas` 标签
- **资源注入**：使用 `hexo.extend.filter.register` 在页面生成后注入CSS和JS资源
- **动态生成**：根据标签参数动态生成画布HTML结构

#### 2. canvas-doodle.js（前端逻辑）

- **画布初始化**：自动初始化页面中所有的涂鸦画布
- **绘图功能**：
  - 支持鼠标和触摸事件
  - 黑色画笔，线宽3px
  - 圆角线条连接
- **存储功能**：
  - 使用 `localStorage` 存储画布数据
  - 数据以Base64编码的PNG图片格式保存
  - 每个画布使用独立的存储键

#### 3. canvas-doodle.css（样式文件）

- **容器样式**：浅灰色背景，圆角边框
- **画布样式**：白色背景，黑色边框
- **按钮样式**：
  - 清除按钮：红色
  - 保存按钮：绿色
  - 加载按钮：橙色
- **状态提示**：操作后显示3秒的状态信息

### 工作原理

1. **标签解析**：Hexo在渲染文章时解析 `{% canvas %}` 标签
2. **HTML生成**：插件生成包含画布和控制按钮的HTML结构
3. **资源注入**：CSS和JS自动注入到页面的 `<head>` 和 `<body>` 末尾
4. **前端初始化**：页面加载后自动初始化所有画布
5. **交互处理**：
   - 用户在画布上涂鸦
   - 点击保存按钮将画布内容转为Base64存储到localStorage
   - 点击加载按钮从localStorage读取并恢复画布内容

## 注意事项

1. **数据存储**：涂鸦内容保存在浏览器本地存储中，清除浏览器数据会丢失
2. **跨设备**：由于使用localStorage，不同设备之间无法同步
3. **画布大小**：建议根据文章布局设置合适的画布尺寸
4. **兼容性**：支持所有现代浏览器，包括Chrome、Firefox、Safari、Edge等

## 开发说明

### 本地测试

1. 在Hexo博客目录下创建 `node_modules/hexo-canvas-doodle` 文件夹
2. 将所有插件文件复制到该文件夹
3. 在文章中添加 `{% canvas %}` 标签
4. 运行 `hexo server` 启动本地服务器测试

### 自定义样式

如果需要自定义样式，可以在博客的主题CSS文件中覆盖默认样式：

```css
.hexo-canvas-doodle-container {
  /* 自定义容器样式 */
}

.hexo-canvas-doodle {
  /* 自定义画布样式 */
}

.hexo-canvas-btn {
  /* 自定义按钮样式 */
}
```
